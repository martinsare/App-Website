# Android App Portal MVP

## Goal

Build a web app where I can upload my Android APK apps, allow users to download them, show update history, and automatically notify my Telegram group whenever I publish a new app or update.

## Tech Stack

* Frontend: React + Vite or Next.js
* Backend: Supabase
* Database: Supabase Postgres
* Auth: Supabase Auth
* File Storage: Supabase Storage
* Hosting: Vercel
* Notifications: Telegram Bot API
* Optional Backend Logic: Supabase Edge Functions

## Main User Roles

### Public User

Can:

* View available apps
* Open app details
* Download APK files
* View version history
* Read install guide
* See changelog / what is new

### Admin

Can:

* Log in
* Add new app
* Upload APK file
* Upload app icon
* Upload screenshots
* Add app description
* Add version number
* Add changelog
* Publish or unpublish apps
* Upload new versions
* View download analytics

## Telegram Integration

When admin uploads a new app or new version, the system should automatically send a message to my Telegram group.

Telegram message should include:

* App name
* Version number
* Changelog
* Download link
* Release date
* Optional app icon or screenshot

Example Telegram message:

New Android App Update Available

App: Department Connect
Version: 1.0.2
What’s new: Fixed login issue and improved dashboard speed.
Download: [APK download link]

## Download Tracking

Every time someone clicks the download button, save a download record.

Track:

* App ID
* Version ID
* Download date/time
* User device/browser if available
* IP address if available
* Referrer if available

Use this to show analytics in the admin dashboard.

## Main Flow

### Upload New App

1. Admin logs in
2. Admin creates app profile
3. Admin uploads icon and screenshots
4. Admin uploads APK
5. Admin adds version number and changelog
6. Admin publishes app
7. Telegram group receives notification
8. App appears publicly on website

### Upload App Update

1. Admin logs in
2. Admin selects existing app
3. Admin uploads new APK
4. Admin enters new version number
5. Admin adds changelog
6. System marks it as latest version
7. Telegram group receives update notification
8. Users can download the new APK

### Download App

1. User opens app page
2. User clicks download
3. System records download
4. APK starts downloading
5. Admin analytics updates

## Important Notes

This platform is for Android APK downloads only.

Users may see Android security warnings because the app is installed outside Google Play Store. The website should look professional and trustworthy, with clear app details, screenshots, version history, and install instructions.

Later, when there is enough money, the apps can be published on Google Play Store. This website can still remain as a portfolio, support, changelog, and direct download page.
